Snapshot com EBS

Anotações

  • O Amazon Elastic Block Store (EBS) é um serviço para fornecer armazenamento em bloco fiável (também conhecido com volumes ou discos rígidos). Foi concebido para ser utilizado com instâncias do Amazon Elastic Compute Cloud (EC2).
  • Quando falamos sobre infraestrutura, isto tem a ver com o disco de servidor virtual. Vale lembrar que quando falamos de infraestrutura estamos falando do modelo de Cloud, nesse caso IAAS.
  • O snapshot do EBS é um serviço de backup nativo do AWS que faz backup dos volumes do EBS em um determinado momento. E é possível configurar a frequência com que eles são tirados.

Armazenamento dos snapshots

  • Os instantâneos do EBS armazenam os volumes do EBS no AWS S3, em uma matriz de armazenamento diferente de onde estão os volumes do EBS.
  • Para fins de recuperação de desastres (DR), os instantâneos do EBS podem ser armazenados em uma região remota.
  • Podemos também otimizar o armazenamento em SSD para operações de leitura/gravação com uso intensivo de I/O.

Quanto custam os snapshots?

  • Os snapshots são oferecidos a diferentes custos em diferentes regiões:
    • Na região leste dos EUA, por exemplo, os snapshots são cobrados a um custo fico de US$0,05 por GB por mês.
    • Na região da UE (Londres), os snapshots são cobrados a US$0,053 por GB por mês.

Qual a diferença entre imagem e o snapshot?

  • No Amazon EC2, uma imagem da máquina da Amazon (AMI) faz o backup de um servidor inteiro, incluindo todos os volumes EBS anexados.
  • Já o snapshot são cópias em pontos no tempo de um volume do Amazon EBS, armazenadas no Amazon S3. Eles podem ser usados para criar um volumes do Amazon EBS, aumentar a durabilidade de dados e fornecer um mecanismos de backup e restauração para volumes do EBS.
